# App UI screenshots for the website

The **See it in action** section uses these two files:

| Website file | Purpose |
|---|---|
| `assets/img/screenshot-start.png` | App window — folder selection / start |
| `assets/img/screenshot-results.png` | App window — scan results |

## Not the same as `instagram/captures/`

`instagram/captures/` holds **marketing slides** (full page sections at 1080×1350, etc.)
generated by `scripts/capture_instagram_screenshots.py` for Instagram posts.

Those files are **not** used on the website. The website reads only
`assets/img/screenshot-start.png` and `assets/img/screenshot-results.png`.

## How to update

1. Take two PNG screenshots from the **desktop app** (macOS or Windows).
2. Save them here as:

```
assets/img/app-screenshots/screenshot-start.png
assets/img/app-screenshots/screenshot-results.png
```

3. Run sync (copies into `assets/img/` for the site):

```bash
bash scripts/sync_app_screenshots.sh
```

Windows:

```cmd
scripts\sync_app_screenshots.bat
```

4. Rebuild and deploy:

```cmd
scripts\build_dist_web.bat
```

5. Optionally refresh Instagram marketing captures **after** step 3:

```cmd
python3 -m http.server 5173
python3 scripts\capture_instagram_screenshots.py
```

Recommended size: ~1024 px wide, PNG, app window only (no browser chrome).
