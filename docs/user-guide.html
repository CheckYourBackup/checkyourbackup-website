User guide

Everything you need to analyze a folder and read your results.

Installation

Download the build for your system from the Download section, unzip it, and run the app. No Python or admin rights are required.

First launch

The builds are not code-signed yet, so your system shows a one-time prompt:

  • macOS: double-click, click OK on the warning (do not move to Trash), then go to System Settings → Privacy & Security and click Open Anyway.
  • Windows: if SmartScreen appears, click More info → Run anyway.

The workflow

  1. Select a mode — Integrity Check or Backup Validation.
  2. Select your folder(s) — the folder to analyze (and the backup folder in validation mode).
  3. Run the analysis — watch the live progress bar.
  4. Review the results — per-file details, summary and status legend.
  5. Open the report — the generated PDF (and CSV).

Mode 1 — Integrity Check

Scans a single folder and classifies every supported file:

  • Healthy — opened and verified.
  • Corrupted — damaged / undecodable.
  • Suspicious — opened with warnings.
  • Unreadable — could not be read.
  • Skipped — intentionally excluded.

Mode 2 — Backup Validation

Compares an original folder with its backup (size first, then SHA-256 hash):

  • Validated — identical in both.
  • Missing in Backup — present in the original, absent in the backup.
  • Different Size — size mismatch.
  • Different Content — same size but different hash.
  • Extra in Backup — present only in the backup.

You also get an overall Backup Health %.

Your reports

  • A corporate PDF report with summary, statistics, a corrupted-files table and per-type sections (images, RAW, ZIP, PDF, Office).
  • A CSV export with: File, Status, Reason, Size, Extension.
  • A dedicated Backup Validation PDF with the health score.
Reports are saved directly into the folder you analyzed. Use the Open Report button to open the PDF straight away.

Good practice

  • Don't assume your backups are healthy — verify them regularly.
  • Re-run a check after large transfers or disk migrations.
  • Keep the PDF/CSV report alongside your archive as proof of its state.
