User guide

Everything you need to analyze a folder and read your results.

Installation

Download the build for your system from the Download section, unzip it, and run the app. No Python or admin rights are required.

First launch

The builds are not code-signed yet, so your system shows a one-time prompt:

  • macOS: double-click, click OK on the warning (do not move to Trash), then go to System Settings → Privacy & Security and click Open Anyway.
  • Windows: if SmartScreen appears, click More info → Run anyway.

The workflow

  1. Select a folder — the archive, backup copy or NAS folder you want to verify.
  2. Run the analysis — watch the live progress bar as each file is opened read-only.
  3. Review the results — per-file details in the table, plus objective counts in the summary.
  4. Open the report — the generated PDF (and CSV) saved in the folder you analyzed.

File statuses

Every supported file in the folder is classified as one of:

  • Healthy — opened and verified.
  • Corrupted — damaged / undecodable.
  • Suspicious — opened with warnings.
  • Unreadable — could not be read.
  • Skipped — intentionally excluded (system, hidden, temporary or report files).

The app shows objective counts only — files analyzed, Healthy, Corrupted, Suspicious, Skipped and data analyzed. There are no scores or ratings.

Info menu

Use Info in the menu bar for quick help without leaving the app:

  • About Check Your Backup — version, website and support.
  • Quick Start — the four-step workflow in one place.
  • What is a Corrupted File? — plain-language examples of damage to look for.
  • Read-Only Analysis — confirmation that your files are never modified.
  • Support Development and Visit Website — open in your browser.

Your reports

  • A corporate PDF report with summary, objective statistics, a corrupted-files table and per-type sections (images, RAW, ZIP, PDF, Office).
  • A CSV export with: File, Status, Reason, Size, Extension.
  • A read-only notice on the PDF cover: Verify before you trust your backup.
Reports are saved directly into the folder you analyzed. Use the Open Report button to open the PDF straight away.

Good practice

  • Don't assume your backups are healthy — verify them regularly.
  • Run a check on each backup copy or archive, not only the original folder.
  • Re-run a check after large transfers or disk migrations.
  • Keep the PDF/CSV report alongside your archive as proof of its state.
